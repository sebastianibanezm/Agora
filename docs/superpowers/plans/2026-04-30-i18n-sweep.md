# i18n Sweep — Hardcoded String Elimination Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire all 21 components with hardcoded strings through next-intl so the app renders fully in Spanish or English based on the `AGORA_LOCALE` cookie.

**Architecture:** Extend `messages/en.json` and `messages/es.json` with 4 new namespaces and extensions to 4 existing ones, then replace every hardcoded string in the affected components with the correct `t()` call. Server components (no `'use client'`) use `async` + `await getTranslations('ns')` from `next-intl/server`. Client components use `useTranslations('ns')` from `next-intl`. Tests that render converted server components get a `vi.mock('next-intl/server', ...)` that returns actual `en.json` values so existing assertions still pass.

**Tech Stack:** Next.js 16 App Router, next-intl v4.11.0, Vitest + @testing-library/react, TypeScript strict

**Spec:** `docs/superpowers/specs/2026-04-30-i18n-sweep-design.md`

Add after the existing imports, before any `describe` block:

```ts
import en from '../messages/en.json';
import { vi } from 'vitest';

vi.mock('next-intl/server', () => ({
  getTranslations: async (ns?: string) => {
    const nsData = ns ? (en as Record<string, unknown>)[ns] : en;
    function resolve(obj: unknown, key: string): string {
      const parts = key.split('.');
      let cur: unknown = obj;
      for (const p of parts) cur = (cur as Record<string, unknown>)?.[p];
      return typeof cur === 'string' ? cur : key;
    }
    return (key: string, values?: Record<string, unknown>) => {
      const val = resolve(nsData, key);
      if (!values) return val;
      return val.replace(/\{(\w+)\}/g/g, (_: string, k: string) => String(values[k] ?? k));
    };
  },
}));
```
